# ROADMAP — Hookline, 8 недель

Исполнители: **Лид** (очередь, воркеры), **Ст** (ingest, delivery, breaker), **A** (signing, backoff, matcher), **B** (API, дашборд), **C** (attempts, sink, метрики, доки).
Уровни: L0 · L1 · L2 · L3 (конкурентность) · L4.
Строка = 1–3 issue на доске GitHub Projects. Неделя закрыта только по Exit-критерию.

Особенность этого роадмапа: модули новичков **не зависят от ядра** и пилятся параллельно с ним с первого дня — никто никого не ждёт.

## Фаза 1 — Приём событий, чистые модули, первая доставка (недели 2–3)

Цель: событие принято, размножено по подпискам, доставлено (пока одним воркером, без ретраев).

| Задача | Кто | Ур. |
|---|---|---|
| queue: claim/ack/nack на SKIP LOCKED + лизы; интеграционный тест «2 воркера не берут одно сообщение» | Лид | L3 |
| worker: пул из N горутин, цикл claim→deliver→ack, `Clock`, graceful shutdown | Лид | L3 |
| ingest: POST /ingest/{app} — валидация, Idempotency-Key, запись event | Ст | L2 |
| ingest: фан-аут события в messages по подпискам (в одной транзакции с событием) | Ст + A | L2 |
| delivery: HTTP-клиент — таймаут 10s, лимит чтения тела 1МБ, классификация ответов (2xx / 4xx / 5xx / сетевая ошибка) | Ст | L2 |
| **signing**: HMAC-подпись и верификация, timestamp-защита, constant-time сравнение, тестовые векторы | A | L2 |
| **backoff**: экспонента + full jitter, cap, детерминированные тесты с фиксированным rand | A | L1 |
| **matcher**: `push`, `order.*`, `*` — чистая функция + таблица кейсов | A | L1 |
| Admin API: apps, endpoints, subscriptions (CRUD), API-ключи, единый формат ошибок | B | L2 |
| HTTP-каркас: chi, recover, request_id, slog | B | L1 |
| **cmd/sink**: приёмник, который по query-параметру отвечает 200/500/таймаутом и логирует полученное — нужен всем для тестов | C | L1 |
| attempts: запись попытки в одной транзакции с ack/nack; GET /messages/{id} с таймлайном | C | L2 |
| openapi.yaml + коллекция Bruno | B | L1 |
| Семинар №1 (Ст): «слои, интерфейсы, ошибки, Clock — на нашем коде» | Ст | — |

**Exit:** `curl` в ingest → sink получает подписанный запрос → в дашборде/API видна успешная попытка. Тест «2 воркера, 100 сообщений, каждое доставлено ровно раз» зелёный с `-race`.

## Фаза 2 — Надёжность: ретраи, DLQ, replay, дашборд (недели 4–5)

Цель: сервис переживает падение получателя и не теряет ничего.

| Задача | Кто | Ур. |
|---|---|---|
| worker: nack с переносом — `next_attempt_at = now + backoff(attempt)`, инкремент попытки | Лид | L3 |
| **reaper**: отдельная горутина, возвращает зависшие `in_flight` (истёк `locked_until`) в `pending` | Лид (+пара со Ст) | L3 |
| Тест устойчивости: убиваем воркера посреди доставки (kill контейнера) — сообщение доезжает другим | Лид + Ст | L3 |
| DLQ: исчерпаны попытки → `status=dead`; выборка GET /messages?status=dead | Ст | L1 |
| Replay: POST /messages/{id}/replay — новое сообщение со ссылкой на исходное | Ст + B | L2 |
| Дашборд: лента событий, детали сообщения с таймлайном попыток, кнопка Replay | B | L2 |
| Дашборд: страница верификации подписи (payload+secret+signature → валидна/нет) | B + A | L1 |
| Пагинация событий и попыток (keyset) | C | L2 |
| Хранение снипета ответа (обрезка 1КБ, редактирование секретов из хедеров в логах) | C | L1 |
| Нагрузочный прогон: 500 сообщений/мин на 3 воркерах, pending не растёт | Лид | L3 |
| Семинар №2 (Лид): «горутины, каналы, context, graceful shutdown — на воркере» | Лид | — |

**Exit:** гасим sink → сообщения копятся и ретраятся с растущими интервалами; поднимаем sink → всё доезжает; принудительно исчерпанные попытки лежат в DLQ и реплеятся кнопкой. Всё видно в дашборде.

**Контрольная точка отвала:** осталось ≤3 человека → режем circuit breaker, rate-limit, Telegram; фаза 3 сокращается до масштабирования + метрик.

## Фаза 3 — Масштаб, защита получателей, dogfooding (недели 6–7)

Цель: 3 реплики, живые вебхуки GitHub и красивое демо.

| Задача | Кто | Ур. |
|---|---|---|
| compose: 3 реплики воркера; тест «3 воркера × 200 сообщений — ровно один раз каждое» | Лид | L3 |
| Rate limit на эндпоинт (token bucket в БД/памяти воркера, честный при N репликах — ADR-0005) | Лид + A | L3 |
| **Circuit breaker**: closed/open/half-open в таблице endpoints, сброс через API | Ст | L3 |
| Воркер-уборщик: удаление событий старше N дней батчами (первая L3-задача A) | A (+пара с Лидом) | L3 |
| `POST /ingest/github/{app}`: верификация `X-Hub-Signature-256`, маппинг `X-GitHub-Event` → event_type | A + Ст | L2 |
| **Dogfooding**: подключить вебхук нашего репозитория к Hookline; подписчики — sink и Telegram-бот | C (+B) | L2 |
| Telegram-подписчик: маленький сервис-приёмник, кидающий сообщение в чат (первая L3-задача C — ретраи/таймауты) | C | L3 |
| Метрики Prometheus: pending, in_flight, dead, возраст старейшего pending, длительность доставки, коды ответов; дашборд Grafana | C + B | L2 |
| Дашборд: блок статуса эндпоинтов (circuit open/closed) — первая L3-задача B (гонки в обновлении статуса) | B (+пара со Ст) | L3 |
| Graceful shutdown под нагрузкой: SIGTERM → доработать in-flight, вернуть остальное в pending | Лид | L3 |
| Каждый из A/B/C закрыл ≥ 1 задачу L3 | все | L3 |

**Exit:** пуш в репозиторий → сообщение в Telegram, прошедшее через нашу очередь; в Grafana видно всплеск; убийство одного из трёх воркеров в этот момент ничего не ломает (это и есть демо-гифка).
