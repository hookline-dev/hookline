# Онбординг: поднять Hookline локально за 30 минут

Цель этого файла — чтобы ты с нуля запустил проект у себя на компьютере и дошёл до первой задачи. Если на любом шаге застрял дольше 15 минут — это не твоя вина, а пробел в этой инструкции: напиши в `#blockers`, мы поможем и допишем сюда решение (см. раздел «Частые проблемы»).

Перед этим файлом должен быть пройден `docs/GIT_FOR_BEGINNERS.md` (аккаунт, git, SSH-ключ, клонирование репозитория).

---

## Что нужно установить

| Инструмент | Зачем | Где взять / как проверить |
|---|---|---|
| **Go 1.26+** | язык, на котором пишем | go.dev/dl → проверка: `go version` |
| **Docker + Docker Compose** | поднимает базу и сервисы одной командой, без ручной установки Postgres | docker.com/products/docker-desktop → проверка: `docker --version` и `docker compose version` |
| **make** | запускает готовые команды проекта короткими словами | Linux/macOS обычно уже есть; Windows — идёт с Git Bash или через WSL. Проверка: `make --version` |
| **git** | работа с кодом | уже установлен на прошлом шаге |

> Docker Desktop должен быть **запущен** (иконка кита в трее/меню). Если он выключен, любая команда `docker ...` и `make up` выдадут ошибку «cannot connect to the Docker daemon».

## Шаги запуска

Все команды выполняются внутри папки проекта (`cd hookline`).

**1. Скопируй файл настроек.**
```bash
cp .env.example .env
```
`.env.example` — образец с настройками (адрес базы, порты и т.п.). Команда делает из него твой личный `.env`. Его править не нужно — значений по умолчанию хватает. Коммитить `.env` нельзя (он уже в `.gitignore`).

**2. Подними инфраструктуру.**
```bash
make up
```
Эта команда через Docker поднимет базу данных PostgreSQL, сервис `hookline`, воркеры и тестовый приёмник `sink`. Первый раз будет качать образы — пара минут, это нормально. Готово, когда в выводе перестанут бежать строки и вернётся приглашение терминала.

**3. Создай таблицы в базе (миграции).**
```bash
make migrate
```
«Миграция» — это применение SQL-скриптов, которые создают нужные таблицы. Без этого база пустая и приложение работать не будет.

**4. Прогони демо-сценарий.**
```bash
make demo
```
Команда создаст тестовое приложение, подписчика и отправит через систему одно событие — чтобы ты сразу увидел, что доставка работает.

**5. Открой дашборд.**
Перейди в браузере на **http://localhost:8080** — там должна быть видна доставленная попытка из шага 4.

**6. Убедись, что тесты проходят.**
```bash
make test
```
Все тесты должны быть зелёными. Это же ты будешь запускать перед каждым PR.

Если все шесть шагов прошли — поздравляю, окружение готово. 🎉

## Полезные команды на каждый день

```bash
make up            # поднять всё
make down          # погасить всё
make logs          # смотреть логи сервисов (Ctrl+C — выйти)
make run           # запустить приложение локально (без docker), для быстрой отладки
make test          # тесты с детектором гонок
make lint          # проверка стиля кода (то же, что проверяет CI)
make migrate       # применить миграции
make seed          # набить базу демо-данными
make help          # показать все доступные команды с описаниями
```

Не помнишь команду — просто набери `make help`.

## Частые проблемы

Этот раздел наполняется по мере того, как каждый проходит онбординг (это задача **T-02** из `STARTER_TASKS.md`). Столкнулся с проблемой, которой тут нет, — реши её и допиши сюда PR-ом. Так ты и себе поможешь, и следующему новичку.

**`make: command not found` (обычно Windows).**
make не установлен или ты не в Git Bash. Открой Git Bash (не cmd/PowerShell) или поставь make через пакетный менеджер / используй WSL.

**`docker: cannot connect to the Docker daemon`.**
Docker Desktop не запущен. Запусти его, дождись, пока иконка перестанет мигать, повтори команду.

**`port is already allocated` / порт занят.**
Какая-то программа уже слушает нужный порт (часто — ранее не погашенный запуск). Сделай `make down`, при необходимости перезапусти Docker Desktop, потом `make up`.

**`make migrate` падает с ошибкой подключения к базе.**
База ещё не успела подняться после `make up`. Подожди 10–15 секунд и повтори. Если не помогло — `make down && make up`, дай базе стартовать, потом миграции.

**Порт 8080 открывается, но страница пустая / ошибка.**
Скорее всего, пропущен `make migrate` или `make demo`. Пройди шаги 3–4 заново.

## Что дальше

1. Прочитай `CONTRIBUTING.md` (правила разработки) и `docs/GITHUB_PROJECTS_GUIDE.md` (как устроена доска и наши ритуалы), если ещё не.

3. Назначь её на себя, переведи в **In Progress** и — по циклу из `docs/GIT_FOR_BEGINNERS.md` — доведи до первого PR.
4. Добавь себя в `docs/team.md`.

Цель первой недели простая: один смерженный PR, любого размера. Даже правка опечатки считается — важно один раз пройти весь путь.
