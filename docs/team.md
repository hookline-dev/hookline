# Команда Hookline

Здесь мы знакомимся и фиксируем, кто за что отвечает. Добавить себя сюда — это часто **первая задача** новичка (T-01): отличный способ впервые пройти весь путь ветка → PR → ревью → мерж на изменении, которое невозможно сломать.

## Как добавить себя (пошагово)

1. Создай ветку: `git switch main && git pull && git switch -c docs/<номер>-add-me-to-team`
2. Впиши свою строку в таблицу «Состав» ниже (скопируй строку-образец и заполни).
3. Если хочешь — добавь абзац о себе в раздел «О нас».
4. Сохрани, закоммить, запушь, открой PR (подробности — в `docs/GIT_FOR_BEGINNERS.md`).
5. В описании PR напиши `Closes #<номер своей задачи>`.

Не знаешь свой часовой пояс в формате UTC — загугли «мой часовой пояс UTC» (например, Москва — UTC+3).

## Кто за что отвечает (расшифровка ролей)

- **Тимлид / архитектор** — ядро (очередь и воркеры), финальное ревью и мерж, архитектурные решения, нарезка задач.
- **Бэкендер** (стажёр) — приём событий, HTTP-доставка, circuit breaker; второй ревьюер для простых задач.
- **Разработчик — чистая логика** — модули без базы и сети (подпись, backoff, матчинг): идеальны для тестов, растёшь от функций к конкурентности.
- **Разработчик — API и дашборд** — REST-эндпоинты, middleware, веб-интерфейс.
- **Разработчик — данные и инструменты** — журнал попыток, тестовый приёмник, метрики, документация.

Полное описание — в `docs/TZ.md`.
